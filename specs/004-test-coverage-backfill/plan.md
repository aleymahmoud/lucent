# Implementation Plan: Test Coverage Backfill + Forecast Benchmarks

**Branch**: `004-test-coverage-backfill` | **Date**: 2026-04-17 | **Spec**: [spec.md](./spec.md)
**Input**: Feature specification from `/specs/004-test-coverage-backfill/spec.md`

## Summary

Bring backend test coverage from 37 tests (forecasting utilities only) to roughly 150-200 tests spanning every service and endpoint, plus a scripted forecast benchmark against 20 synthetic datasets. Deliver in three priority tiers: P1 = critical forecasting pipeline + residual-persistence contract tests, P2 = preprocessing + endpoints + exporters + benchmark, P3 = coverage threshold and CI-friendly separation.

## Technical Context

**Language/Version**: Python 3.11. No frontend work.
**Primary Dependencies**: pytest (installed), pytest-asyncio (installed). New dev deps: `pytest-cov`, `httpx` (for ASGI test client), `fakeredis` (optional вЂ” for integration tests without real Redis).
**Storage**: Tests use the existing Neon Postgres with per-test transaction rollback (SQLAlchemy async savepoint pattern). Redis tests use `fakeredis` or are marked `integration` and skipped when `TEST_REDIS_URL` is missing.
**Testing**: pytest. Markers: default (unit), `integration` for endpoint + DB + Redis tests. Coverage via `pytest --cov`.
**Target Platform**: Windows Server 2022 dev, Linux container CI.
**Project Type**: Backend monorepo component.
**Performance Goals**: Unit tests complete in < 60 s. Integration tests complete in < 3 min. Benchmark completes in < 10 min.
**Constraints**: No new runtime dependencies. Test-only deps are acceptable but must be small and pure-Python. Prophet-dependent tests MUST skip cleanly when CmdStan is missing (xfail or skip).
**Scale/Scope**: ~12 new test files, 1 new benchmark script, 1 new conftest.py for fixtures. ~2000 lines of test code total across all tiers.

## Constitution Check

- **Discovery Before Creation**: No new shared utilities. Test fixtures are scoped per-file unless reused, in which case they go in a shared `conftest.py`.
- **Alembic-managed schema**: Tests must not mutate schema вЂ” if a test needs a fixture table, it uses a factory or transaction rollback.

**Gate status**: PASS.

## Phase 0 вЂ” Research & Decisions

Five open questions resolved in [research.md](./research.md):

1. **Per-test DB isolation**: savepoint rollback vs fresh schema. Decision: async savepoint per test inside an outer transaction.
2. **Fake Redis or real Redis**: decision: `fakeredis` for unit tests; real Redis for `integration` marker only.
3. **Prophet skip marker**: decision: pytest `@pytest.mark.skipif(not cmdstan_available())` at the file level.
4. **Synthetic dataset library**: decision: handwritten numpy generators in `tests/data/synthetic.py` вЂ” full control, zero deps.
5. **Coverage target**: decision: 70% for `forecasting/`, 50% for `services/`. Not a gate вЂ” a report.

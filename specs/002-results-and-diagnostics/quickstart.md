# Quickstart — Manual Verification

**Feature**: Results & Diagnostics Parity
**Plan**: [plan.md](./plan.md)

## Purpose

Walk through each user story end-to-end after implementation. Each walkthrough maps to one or more user stories in [spec.md](./spec.md).

## Prerequisites

1. Backend + frontend running (`pm2 restart lucent-backend lucent-frontend`).
2. Test datasets from spec 001's quickstart (generated by `scripts/generate_test_datasets.py`).
3. Logged in as an analyst with access to a tenant.

## Walkthrough 1 — Cross-validation visible on Results (US1)

**Dataset**: `daily_90.csv` (90 rows, daily).

Steps:
1. Forecast tab: select dataset, pick the only entity, method = ARIMA, horizon = 14.
2. Enable Cross-Validation: 3 folds, rolling, train size 70%.
3. Run forecast.
4. When status flips to Completed, navigate to Results.

**Expected**:
- Metrics cards at top show in-sample MAE/RMSE/MAPE.
- A new "Cross-Validation" section below shows:
  - Method: rolling
  - Folds: 3
  - Per-fold rows: Fold 1, Fold 2, Fold 3 with MAE/RMSE/MAPE values
  - Average row highlighted in bold
- If CV wasn't enabled, the section is absent (not a placeholder).

## Walkthrough 2 — Real residuals drive diagnostics (US2)

**Dataset**: `daily_90.csv`.

Steps:
1. Run any forecast.
2. Open Diagnostics → Residuals tab.
3. Check the browser dev-tools network response for `/diagnostics/{id}/residuals`.

**Expected**:
- `is_synthetic: false` in the JSON response.
- Residuals array length roughly equals training data size (not a fixed 100).
- Ljung-Box p-value varies across different datasets (not suspiciously always ~0.5).

**Negative test**: Query a forecast from before this feature ships (an old record in DB).
- `is_synthetic: true` → UI shows the "historical record" banner.

## Walkthrough 3 — All residual plot types render (US3)

**Dataset**: `daily_90.csv`, run ARIMA.

Steps:
1. Open Diagnostics → Residuals tab.
2. Use the plot-type selector to cycle: Time Series → Histogram → QQ → ACF.

**Expected**:
- **Time Series**: scatter + line, with dashed reference lines at 0, ±σ, ±2σ.
- **Histogram**: bars with overlaid normal curve (blue) and KDE (red).
- **QQ**: points with diagonal reference line.
- **ACF**: bars with confidence bands at ±1.96/√n.

Tests box below the plot shows Ljung-Box, Breusch-Pagan, Shapiro-Wilk with p-values and plain-English interpretation.

## Walkthrough 4 — Coefficient significance shown (US4)

**Dataset**: `daily_90.csv`, run ARIMA (auto).

Steps:
1. Diagnostics → Model tab.

**Expected**:
- Coefficients table columns: Name, Estimate, Std. Error, Z-stat, P-value.
- Rows with p-value > 0.05 show a "Not significant" badge.

**Prophet case**:
1. Switch method to Prophet, re-run.
2. Open Diagnostics → Model tab.

**Expected**:
- No coefficient table. Instead, a hyperparameter panel shows changepoint prior, seasonality prior, seasonality toggles, growth type.

## Walkthrough 6 — Model comparison UI (US6)

Steps:
1. After running any single-entity forecast, open Diagnostics.
2. Click "Compare Models" button.
3. Check ARIMA + ETS (leave Prophet unchecked), click "Run Comparison".

**Expected**:
- Loading indicator while comparison runs.
- Table appears with columns: Model, MAE, RMSE, MAPE, Composite Score.
- Best row highlighted (green background).

**Edge case**: include Prophet when CmdStan is missing. Expected: Prophet row shows "error: CmdStan not available" but ARIMA + ETS still complete.

---

## Walkthrough 7 — Chart controls (US7)

Steps:
1. Results page of any forecast.
2. Switch plot type: Line → Line with Points → Area.
3. Uncheck "Show intervals".
4. For a 90+ day dataset, check the range slider at the bottom of the chart.

**Expected**:
- Plot visual changes match the selection.
- Intervals toggle hides/shows the band.
- Range slider scrolls/zooms the visible window.

---

## Walkthrough 8 — Table view modes (US8)

Steps:
1. Results → Data tab.
2. Use the view selector: Forecast Only → Historical + Forecast → Full Data.

**Expected**:
- **Forecast Only** (default): Date, Forecast, Lower, Upper.
- **Historical + Forecast**: adds Historical column; rows span both ranges.
- **Full Data**: Date, Type (Historical/Forecast), Value, Lower, Upper.

## Walkthrough 9 — Excel export (US9)

### 9a. Single entity

Steps:
1. Results page, click "Download → Excel".

**Expected**:
- `.xlsx` file downloads.
- Open it: sheets include "Forecast", "Metrics", "Model Summary", and "Cross-Validation" (if CV was enabled).

### 9b. Batch

Steps:
1. Run a batch forecast on 3+ entities.
2. Click "Download All Entities (Excel)".

**Expected**:
- `.xlsx` with "Summary", "All Data", and one sheet per entity (names truncated to 31 chars if long).

---

## Walkthrough 10 — Diagnostics export (US10)

Steps:
1. Diagnostics page, click "Export → HTML".

**Expected**:
- HTML file downloads. Open in browser — contains:
  - Header with entity + method + date range
  - In-sample + CV metrics
  - Residual plots (embedded)
  - Test results
  - Quality scores

### PDF case (if weasyprint or equivalent is installed)

Steps:
1. Click "Export → PDF".

**Expected**:
- `.pdf` downloads, same content as HTML.
- If lib missing: toast error "PDF export unavailable; use HTML".

- 20 synthetic datasets tested.
